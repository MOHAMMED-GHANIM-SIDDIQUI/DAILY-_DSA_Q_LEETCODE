# 38. Count and Say

## 🧩 Problem Summary
The count-and-say sequence is a series of digit strings defined by a recursive formula. Starting with "1", each subsequent term is generated by describing the previous term (run-length encoding). Given an integer n, return the nth term of the sequence.

### 📌 Constraints
- `1 <= n <= 30`

---

## 💭 Intuition
👉 Each term is simply the run-length encoding of the previous term. Count consecutive identical characters and concatenate the count followed by the character.

---

## ⚡ Approach — Iterative Simulation

### 🧠 Idea
- Start with "1".
- For each iteration up to n-1, scan the current string.
- Count consecutive identical characters.
- Build the next string by appending `count + character`.
- Use `std::move` for efficient string transfer.

---

## 💻 Code

```cpp
class Solution {
 public:
  string countAndSay(int n) {
    string ans = "1";

    while (--n > 0) {
      string next;
      for (int i = 0; i < ans.length(); ++i) {
        int count = 1;
        while (i + 1 < ans.length() && ans[i] == ans[i + 1]) {
          ++count;
          ++i;
        }
        next += to_string(count) + ans[i];
      }
      ans = std::move(next);
    }

    return ans;
  }
};
```

## 🧠 Dry Run
### Input
```
n = 4
```
### Steps
```
Start: ans = "1"
Iteration 1 (n=3): "1" -> one 1 -> "11"
Iteration 2 (n=2): "11" -> two 1s -> "21"
Iteration 3 (n=1): "21" -> one 2, one 1 -> "1211"
Result: "1211"
```

---

## ⏱️ Time Complexity
```
O(n * L) where L is the length of the string at each step (grows exponentially, bounded by constraints)
```

## 💾 Space Complexity
```
O(L) — storing the current and next strings
```

---

## ⚠️ Edge Cases
- `n = 1` — return "1" directly without any iteration.
- Large n values — string grows rapidly but n <= 30 keeps it manageable.

## 🎯 Interview Takeaways

- Using `std::move` avoids unnecessary string copies.
- The inner while loop handles consecutive character counting elegantly.
