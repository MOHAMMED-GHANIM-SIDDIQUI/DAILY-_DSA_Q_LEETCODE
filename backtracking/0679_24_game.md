# 679. 24 Game

## 🔗 Problem Link
https://leetcode.com/problems/24-game/

## ⚡ Difficulty
Hard

## 🏷️ Topics
Array, Math, Backtracking

---

## 🧩 Problem Summary
Given an integer array cards of length 4, each card has a value in [1, 9]. Using +, -, *, / and parentheses, determine if you can arrange these four numbers to get the value 24. Division is real-valued (not integer division).

### 📌 Constraints
- `cards.length == 4`
- `1 <= cards[i] <= 9`

---

## 💭 Intuition
👉 Try all possible ways to pick two numbers, apply an operation, and replace them with the result. Recursively reduce the problem from 4 numbers to 3, then 2, then 1, checking if the final result is 24.

---

## ⚡ Approach — DFS with All Operations

### 🧠 Idea
- Pick every pair of numbers from the current list.
- For each pair, generate all possible results (a+b, a-b, b-a, a*b, a/b, b/a).
- Replace the pair with the result and recurse on the smaller list.
- Base case: one number left, check if it equals 24 (with floating-point tolerance).

---

## 💻 Code

```cpp
class Solution {
 public:
  bool judgePoint24(vector<int>& nums) {
    vector<double> doubleNums;

    for (const int num : nums)
      doubleNums.push_back(num);

    return dfs(doubleNums);
  }

 private:
  bool dfs(vector<double>& nums) {
    if (nums.size() == 1)
      return abs(nums[0] - 24) < 0.001;

    for (int i = 0; i < nums.size(); ++i)
      for (int j = 0; j < i; ++j) {
        for (const double num : generate(nums[i], nums[j])) {
          vector<double> nextRound{num};
          for (int k = 0; k < nums.size(); ++k) {
            if (k == i || k == j)  // It is used in `generate()`.
              continue;
            nextRound.push_back(nums[k]);
          }
          if (dfs(nextRound))
            return true;
        }
      }

    return false;
  }

  vector<double> generate(double a, double b) {
    return {a * b, a / b, b / a, a + b, a - b, b - a};
  }
};
```

## 🧠 Dry Run
### Input
```
cards = [4, 1, 8, 7]
```
### Steps
```
Pick (8, 7): generate -> {56, 8/7, 7/8, 15, 1, -1}
  Try result=1: remaining [4, 1, 1]
    Pick (4, 1): generate -> {4, 4, 1/4, 5, 3, -3}
      Try result=4: remaining [1, 4]
        Pick (1, 4): generate -> {4, 1/4, 4, 5, -3, 3}
        None equals 24
      Try result=5: remaining [1, 5]
        Pick (1, 5): generate -> {5, 1/5, 5, 6, -4, 4}
        None equals 24
  ...
  Try result=15: remaining [4, 1, 15]
    ... eventually: 8*(7-4+1) = 8*4 = 32? No.
    (8-4)*(7-1) = 4*6 = 24 ✓
Result: true
```


```
O(1) — fixed recursion depth of 3 levels
```

## ⚠️ Edge Cases
- Division by zero — handled naturally since b/a or a/b with 0 won't produce 24.
- Floating-point precision — using tolerance of 0.001.
- All same numbers — e.g., [8,8,8,8] -> 8/(8-8/8) is not valid, but some combos may work.
- Result requires specific parenthesization.

---

## 🎯 Interview Takeaways
- Exhaustive search is acceptable when the search space is bounded and small.
- Generating all operations for a pair and recursing on smaller lists is a clean approach.
- Floating-point comparison requires epsilon tolerance.

---

## 📌 Key Pattern
👉 **"Reduce N numbers to 1 by trying all pair-operation combinations — bounded exhaustive search"**

## 🚀 Final Thoughts
The 24 Game is a classic exhaustive search problem. Despite seeming complex, the fixed input size (4 cards) makes brute force perfectly viable. The elegance lies in reducing the problem size by one at each recursive step.

---

✨ **Rule to remember:**
> "Four cards, six operations per pair, three reduction steps — brute force is fast enough when the space is bounded."
